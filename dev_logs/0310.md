### 6. 基础设施变更
- 所有任务必须通过 SLURM 提交（`sbatch --nodelist=<node>`），不能直接在节点上运行
- 之前训练中出现的 CUDA illegal memory access（weight sync 阶段）可能是管理员后台 kill 非 SLURM 任务导致，通过 SLURM 正常提交后应不再出现
- Docker 容器环境部署在 research-common-03（10.56.12.57:5000），训练节点通过网络调用，因此训练可提交到任意有 GPU 的节点

### 7. 启动速度优化（已完成）
- **问题**：`uv run --isolated` 导致每个 Ray worker 都独立安装 182 packages，v3 训练中实际安装 600+ 次，启动浪费 ~15 分钟
- **根因**：`--isolated` 创建临时虚拟环境，Ray spawn 的每个 worker（vLLM TP workers、training actors、env workers）都继承此行为并重复安装
- **修复**：SLURM 脚本改为 `source .venv/bin/activate && python -m ...`，直接使用预装好的 venv
- **验证**：venv 中 194 个包，核心依赖版本与 `uv run --isolated` 完全一致（同一个 uv.lock 解析），多出的 12 个是 dev 依赖不影响训练
- 预计启动时间从 ~20 分钟降到 ~3-5 分钟
